# 服务器管理

## 创建服务器配置

1. 在首页点击“添加”，选择“添加服务器”。
2. 填写主机、端口、用户名。
3. 选择认证方式（密码或私钥）。
4. 在服务器弹窗中保存后，执行一次快速连接验证。

## 在服务器弹窗中管理钥匙链

1. 在服务器弹窗的“钥匙链”下拉中选择“创建钥匙链”，可打开钥匙链编辑弹窗。
2. 使用内联凭据时，点击“保存为钥匙链”会打开已预填凭据的钥匙链编辑弹窗。
3. 保存钥匙链后，Cosmosh 会自动将该钥匙链回填到当前服务器表单。
4. 点击“编辑凭据”可在同一弹窗流程中继续编辑当前共享钥匙链。

## 默认备注模板

- 在“设置 > 高级 > 默认服务器备注模板”中配置的内容，会在新建服务器配置时预填到备注字段。
- 该模板仅作用于新建条目，不会覆盖已有服务器备注。

## 服务器地址显示方式

- 可在“设置 > 通用 > 显示完整服务器地址”中控制服务器列表的地址显示方式。
- 开启（默认）：显示完整服务器地址。
- 关闭：地址会以星号脱敏显示，降低肩窥风险。

## 日常整理方式

1. 使用统一命名（例如 `env-region-role`）。
2. 按 `prod`、`staging`、`dev` 等环境分组。
3. 对高风险与低风险主机做明显区分。
4. 刚修改过的主机可临时放入复查分组。

### 标签行为说明

- 在首页查看某个文件夹时，标签筛选仍会显示，且仅展示该文件夹内实际使用的标签。
- 未被任何服务器使用的自定义 SSH 标签会在服务器或标签更新后自动清理。
- 保留标签 `favorite` 不会被自动删除。

## 更新与删除

1. 在首页服务器右键菜单中选择“编辑”。
2. 主机信息或凭据变更后立即更新条目。
3. 编辑后执行一次快速连接验证。
4. 清理过期条目，减少误连风险。
5. 删除生产条目前请再次确认。
